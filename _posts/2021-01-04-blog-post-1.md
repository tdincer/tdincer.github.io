Blending is an ensemble learning technique that combines predictions from a collection of models by weighted averaging or voting. In this post, I’ll show how to find the optimum weights with the method of Lagrange multipliers when the performance metric is differentiable.

For simplicity, let's consider a case that we have 3 models to predict a binary target for N samples where performance of the ensemble is measured by binary cross entropy (a.k.a log loss).

Recall that the binary cross entropy can be written as:


$$
f (y_{ti}, y_{pi}) = - \frac{1}{N} \sum_{i=1}^{N} y_{ti} . log(y_{pi}) + (1-y_{ti}) . log(1-y_{pi}) \label{eq1}
$$

where $y_{ti}$ denotes the actual labels and $y_{pi}$ denotes the predictions.

Then, the binary cross entropy of the blend would be:


$$
f (y_{ti}, w_1 . y_{pi1} + w_2 . y_{pi2} + w_3 . y_{pi3}) \label{eq2}
$$

Since the binary cross entropy $f$ is a convex function, we can use the method of Lagrange multipliers to find the optimum values of the blending weights $w_1$, $w_2$, and $w_3$. Note that the weights have to be positive and sum up to 1.

The Lagrangian for Eq-$\ref{eq2}$ with the aforementioned constraints:


$$
L(y_t, y_p, w, \lambda) = f\left(y_t, \sum_{j=1}^{3} y_{pij}w_j \right) - \lambda \left(1-\sum_{j=1}^3 w_j\right) \label{eq3}
$$

Given Eq-$\ref{eq3}$, the optimum weights can be found by solving:


$$
\nabla_{w,\lambda} L (w, \lambda) = 0 \label{eq4}
$$

Eq-$\ref{eq4}$ is a compact form of 4 equations that are equal to 0, so all 4 unknown variables ($w_1, w_2, w_3, \lambda$) can be obtained by solving them.

## How do we implement this in python?

The implementation can be easily done with the **autograd** and **scipy** libraries in python. The former will help us obtain the partial derivatives, and the latter will solve the system of equations.

Let's start by importing the libraries we will use

```python
from autograd import grad
import autograd.numpy as anp  # autograd likes to work with its own numpy wrapper.
from scipy.optimize import fsolve
```

Next, we will generate some data. We need binary labels for 1000 samples and corresponding binary predictions from 3 different models. To make the results reproducable we will generate the data deterministically.

```python
sample_size = 1000

# Labels
npseeded = anp.random.RandomState(42)
labels = npseeded.randint(2, size=sample_size)  # shape: [smaple_size]

# Predictions
npseeded = anp.random.RandomState(42)
p1 = npseeded.dirichlet([5., 2.], size=[sample_size])  # shape: [smaple_size, 2]
npseeded = anp.random.RandomState(84)
p2 = npseeded.dirichlet([6., 2.], size=[sample_size])
npseeded = anp.random.RandomState(86)
p3 = npseeded.dirichlet([3., 2.], size=[sample_size])

# Predictions stacked
ps = anp.stack([p1, p2, p3]) # shape: [3, sample_size, 2]
```

Now, let's define the binary crossentropy function with numpy and calculate how individual models perform. When defining the binary crossentropy, we will make use of the numpy inside the autograd.

```python
def binarycrossentropy(y_true, y_pred, clip=1e-15):
    y_true = anp.array(y_true)
    y_pred = anp.array(y_pred)
    y_pred = anp.clip(y_pred, clip, 1 - clip)
    loss = anp.where(y_true == 1, -anp.log(y_pred[:, 1]), -anp.log(1 - y_pred[:, 1]))
    return loss.mean()

def individual_cvs(y_true, y_pred):
    for i, y_p in enumerate(y_pred):
        print('M%d metric: %.7f' % (i, binarycrossentropy(y_true, y_p)))  
```

The binary crossentropy for each of our models is as printed below. The best performing model seems to have a binary crossentropy of ~0.831. We will try to get a lower value than this with the optimized weights.

```python
individual_csv(labels, ps)
```

```
M0 metric: 0.8896173
M1 metric: 0.9780689
M2 metric: 0.8311425
```

This is where the things get serious. We will define 3 functions to implement the method of Lagrange multiplier. The first one will calculate the weighted average of the predictions (second argument of Eq-$\ref{eq2}$), the second one will calculate the Lagrangian ($L$) defined in Eq-$\ref{eq3}$, and the third one will calculate the partial derivatives in Eq-$\ref{eq4}$.

```python
def blend_oofs(y_pred, weights):
    return anp.tensordot(y_pred, weights, axes=(0, 0))

def Lagrange_func(params):
    ws = anp.array(params[:-1])
    _lambda = params[-1]
    oof_blend = blend_oofs(ps, ws)
    return binarycrossentropy(labels, oof_blend) - _lambda * (ws.sum() - 1.)

def partial_derivatives(params):
    grad_L = grad(Lagrange_func)
    pars = grad_L(params)
    dLdws = pars[:-1]
    res = anp.append(dLdws, anp.sum(params[:-1]) - 1.)
    return res
```

Solving Eq-$\ref{eq4}$ requires some initial guesses for the weights as well as the Lagrange multiplier ($\lambda$). We will generate the initial guesses randomly from a dirichlet distribution to ensure that the weights add up to 1. Our initial guess for $\lambda$ will be 1. We will make use of scipy's fsolve to solve our equations.

```python
npseeded = anp.random.RandomState(22)
weights = npseeded.dirichlet([2] * len(ps), size=1)[0].tolist() + [1.]
print('Initial parameters (w1, w2, w3, lambda):', weights)
pars = fsolve(partial_derivatives, weights)
```

```
Initial parameters: [0.5064900505541892, 0.13109674512731836, 0.36241320431849255, 1.0]
```

After solving the equations with the line above, we can calculate the binary cross entropy of the blend.

```python
oof_b = blend_oofs(ps, weights[:-1])
print('Blend metric:', binarycrossentropy(labels, oof_b))
print('Optimized Weights (w1, w2, w3, lambda):', pars)
```

```
Blend metric: 0.7974631859476515
Optimized Weights: [ 0.37971505  0.03327863  0.58700631 -0.2154365 ]
```

The blend's binary cross entropy is ~0.797, which is an improvement of 0.034 to our best model's performance!

Now, let's see what would be the binary cross entropy if used equal weights.

```
oof_b = blend_oofs(ps, [1/3., 1/3., 1/3.])
print('Blend metric:', binarycrossentropy(labels, oof_b))
```

```
Blend metric: 0.806171574643176
```

Our optimized weights still yield a better binary cross entropy than the equally weighted case by 0.009. Such an improvement would make a huge difference in a machine learning competition!
